# Operator Agent

The Operator dashboard is Robert's read-only control surface for local agent operations.

Phase 47 adds a Sales Local CRM Queue showing active local opportunities, pending follow-ups, generated drafts, proposal tasks, research queue signals, and proposal queue counts. The Operator view does not write to external CRM, send email, browse websites, or submit proposals.

Phase 48 adds a read-only Sales Research Intelligence panel showing intake queue, pending reviews, duplicate review queue, verification tasks, enrichment progress, rejected sources, approved sources, and confidence trend. Operator does not approve sources automatically or merge duplicates automatically.

Phase 49 adds a read-only Assigned Sales Tasks panel showing Operator handoffs, verification queue, duplicate review queue, missing info queue, proposal prep support queue, and blocked items. Workflow status changes are handled by local CLI commands with audit trails.
# Phase 50 Sales Intelligence Visibility

The Operator dashboard keeps Sales queues read-only and clearer for daily work. Operator can see assigned Sales tasks, verification work, duplicate review queue, missing-info backlog, proposal prep support queue, blocked items, and local CRM metrics.

Phase 50 does not allow Operator to browse, send customer email, sync CRM records, submit proposals, approve Executive gates, approve sources, or merge duplicate opportunities automatically.
