# Engineering Knowledge Graph

The Engineering Knowledge Graph is generated by:

```bash
node scripts/engineering-scan.mjs
```

It writes:

```text
dashboard/public/engineering-graph.json
```

## Repositories

The scanner attempts to index:

- `~/Documents/Vyra-Part-1`
- `/Volumes/Install macOS Sequoia/Vyra Agents`
- `~/Documents/Vyra-Software`
- `~/Documents/vyra-website`

Missing repositories are represented with `missing` status instead of failing the scan.

## Nodes

The graph can contain:

- repository
- folder
- file
- route
- screen
- component
- hook
- service
- api_endpoint
- supabase_function
- migration
- table
- rls_policy
- storage_bucket
- package_dependency
- env_variable_name
- npm_script
- document

## Relationships

The graph can contain:

- contains
- imports
- uses
- defines
- creates_table
- alters_table
- references_table
- calls_function
- depends_on
- has_script
- declares_env
- documents

## Data Safety

The scanner stores metadata only. It does not store file contents. It ignores secret-bearing env files and records only env variable names from safe example files or source-code references.
